Для начала будем реализовывать стратегию 2а - несколько моделей генерируют вопросы, одна модель отвечает

# Генерация вопросов
> Главная идея: \
не давать задачу модели "сгенерируй 3к вопросов по аудиту", а построить таксономию - сетку знаний, где на осях размещены некоторые домены (темы предметной области внутреннего аудита), а на пересечениях получается узкий, конкретный контекст-кейс, под который генерится несколько вопросов

дополнительно:
1) в промт добавлять ссылки на реальные референсные статьи или утвержденные стандарты, чтобы снизить риск галлюцинаций нашей модели-вопросчика + модель будет цепляться именно за стиль оформления аудиторской речи, а не обычной разговорной, на которую она была предобучена
2) должны по-максимуму заставить модель быть разнообразной в плане формулировок, однако это может повысить риск галлюцинации
3) пройтись фильтром дедупликации по смыслу (убираем похожие по косинусной близости эмбеддинги)
4) использовать LLM-as-judge для проверки придержанности темы и наличия галлюцинаций (проверяющая модель должна быть из другого семейства)
5) после чистки заполнить пустые ячейки
